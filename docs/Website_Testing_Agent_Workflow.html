What is the Website Testing Agent?

The Website Testing Agent is an automated quality-assurance tool for websites. Instead of a person manually clicking through every page, checking spelling, resizing the browser, and writing bugs into a spreadsheet, you simply enter a website address and click “Run audit.” The agent opens the site in a real browser and inspects it the way a careful QA tester would.

It answers practical questions automatically, for example:

  • “Are there any broken links or broken images on the site?”
  • “Does the layout break on mobile, tablet, or small screens?”
  • “Are there spelling mistakes, missing page titles, or missing meta descriptions?”
  • “Do the contact forms validate input and keep the data when there’s an error?”

Every problem it finds becomes an issue with a severity, the exact page it’s on, steps to fix it, and a screenshot with the problem area circled in red. The results are shown live, organised page-by-page, and can be exported to Excel or CSV.

2

Why it is “deterministic” (not AI)

Some tools hand the work to an AI model that decides what to do on the fly. This agent does the opposite: it runs a fixed, repeatable set of checks written in code. The same website always produces the same results.

No LLM · No API keys · No rate limits Because there is no AI model in the loop, audits never stall waiting on a quota, cost nothing per run, and are 100% repeatable. What you see is measured directly from the page — never guessed.

How an audit runs (a real example)

You watch each step appear live in the audit stream as the engine works through the site.

Example: auditing a multi-page clinic website.

Step by step

  • Branding & header — loads the home page, checks the logo is visible and links home, the menu exists, the favicon and page title are present.
  • Page crawl — follows the menu to inner pages (About, Services, Contact…), checking each one has a title and a link back home.
  • Broken links & anchors — collects every link on every page and checks each returns a valid response — flagging 404s, dead links, and bad anchors.
  • Content, spelling & fonts — scans copy for spelling and repeated words, checks heading order, and flags inconsistent fonts.
  • Forms & validation — fills the contact form with test data, submits, and checks labels, tooltips, required-field errors, email validation, and that typed data survives an error.
  • Buttons, keyboard & images — checks buttons are a consistent size, not cut off or overlapping, keyboard-reachable, and that images load and have alt text.
  • Resolutions & responsive — resizes to 6 screen sizes (640×480 → 1920×1080, tablet, mobile) and checks for overflow, overlap, missing padding, and horizontal scrollbars.
  • Performance & SEO — measures load time per page, flags oversized images, and checks meta titles, descriptions, and H1 headings.
  • Mark evidence — re-opens each affected page, draws a red box on the exact problem element, and captures a focused screenshot.
Honest by design The agent only reports problems it can actually measure. Checks it genuinely can’t verify automatically (for example, whether a confirmation email was delivered) are left as pending for a human — never guessed or marked as passed.

The test-case toolbox

The engine runs roughly 60 checks across 10 categories on every audit. You never choose them — they all run automatically.

Branding & Header
Logo visible & links home, menu present, header contact info, favicon, page title on every page.
Navigation & Links
Menu items work, home link on every page, no broken links, external links open in a new tab, anchors resolve.
Content, Spelling & Fonts
Spelling, repeated words, no placeholder text, consistent fonts, heading sizes & hierarchy, alignment, lists.
Forms & Validation
Labels, tooltips, required-field errors, email/phone validation, error spelling, data preserved on error, dropdowns.
Buttons & UI Standards
Consistent size & style, text not cut off, not overlapped, keyboard-accessible.
Images & Media
Images load, no broken images, alt text present.
Social Media & Footer
Social links present & valid, open in a new tab, footer links, privacy/terms, contact info & copyright.
Responsive / Resolutions
Layout intact at 6 screen sizes, no horizontal scroll, viewport meta tag, mobile & tablet integrity.
Performance & Usability
Page loads within 3 seconds, images optimised, reasonable page weight.
SEO & Meta Tags
Optimised page title, meta description present, single H1 per page, search-friendly URLs.
Every page, every size Content, SEO and layout checks run on each crawled page — and layout is re-checked at every screen size — so a problem on the “Services” page at mobile width is reported separately from the home page at desktop width.

Evidence & screenshot marking

Every problem that lives on a specific element gets a focused screenshot — the engine scrolls to the element, draws a red box around it, and captures just that part of the page. No more hunting through a full-page image to find what’s wrong.

  • Layout issues are captured at the exact screen size where they break, with the overflowing or overlapping elements boxed.
  • Content, image, link, form, button and footer issues are boxed on the element they affect.
  • Each issue gets its own screenshot — no two issues share the same generic image.
  • Abstract issues (e.g. “fonts differ across pages”, “page weight too heavy”) have no single spot to box, so they’re reported without a screenshot rather than a misleading one.

Manual QA scenarios it covers

The engine was built from a standard manual-QA test sheet. These real-world scenarios are now checked automatically:

  • Content is free of spelling & grammar errors
  • Fonts are consistent as per the design
  • Text is properly aligned
  • Error messages are correct and match field labels
  • Tooltip text is present for every field
  • Fields are properly aligned
  • Adequate spacing between labels, rows & errors
  • Buttons use a standard format and size
  • Clicking the logo loads the home page
  • A home link exists on every page
  • Social media links are valid
  • No broken links or images
  • The site works at multiple resolutions
  • Smooth navigation between pages
  • Pages load within 3 seconds
  • Scrollbar appears only when required
  • Filled data is preserved on a submit error
  • A title displays on every page
  • Fields & buttons are keyboard-accessible
  • Dropdown text is not truncated by field size
